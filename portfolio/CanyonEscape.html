Canyon Escape

Canyon Escape is a short game developed for my Math course to showcase my knowledge of vectors, forces and rotations. You play as a heavy ship trying to make it to the end of an obstacle course. This game was a great experience to showcase the technical skills and math skills I had learned throughout the program.

Movement

The player's ship moves continuously in one direction, similar to an endless runner, and the player is able to move the ship from side to side to avoid obstacles. This was achieved by caching the players inputs and using them to set the rigidbody's velocity on the X axis. A lower speed was chosen to make the spaceship feel heavier, as the model used resembles a tank.
Rotation is also applied to the ship, to make it look like it's leaning in the direction it's moving.

Limited Flight

Since the ship is intended to feel heavier, I gave it limited flight so it can still maneuver over obstacles, but is unable to fly for long periods. This was handled by setting the rigidbody's Y velocity to a fixed value while an input is held, and then running a timer in the background. Once the input is released or the timer hits 0, then the velocity is set back to 0 and flight is disabled until the ship lands.
Rotation is also applied to make the ship tip upwards.

Turret Rotation

As an added feature, and because the model I used was a sci-fi tank, I added turret functionality to blast open metal walls. The challenge with this feature was handling the rotation of the turret independently from the rest of the ship. In order to include this, I separated the turret from the rest of the ship, and created a separate script to handle its rotation. This script responded to the player's inputs for rotating the turret, separately from the ships other functions.
